Shift Left Testing - Overview

Shift Left Testing adalah strategi QA yang melibatkan tim testing lebih awal dalam software development lifecycle (SDLC), sebelum development phase benar-benar selesai.

üìà Perbandingan: Tradisional vs Shift Left

‚ùå Tradisional (Test at End)

  • Testing mulai saat development selesai
  • Bug ditemukan terlambat
  • Rework mahal & time-consuming
  • Risk tinggi di release
  • Kolaborasi minimal

‚úÖ Shift Left (Early Testing)

  • Testing mulai dari requirements phase
  • Bug ditemukan lebih awal
  • Rework minimal & efisien
  • Risk rendah di release
  • Kolaborasi tinggi

Testing Phases

Empat fase testing yang melibatkan QA sejak awal hingga akhir development cycle.

Phase 0

Requirements Analysis

Timeline: Pre-Development

  • Requirements review
  • Risk assessment
  • Test planning
  • Resource allocation

Aktivitas Utama:

  • Review user stories & acceptance criteria
  • Identify gaps & ambiguities
  • Document test scenarios
  • Create test matrix
  • Analyze high-risk features
  • Define test scope
  • Plan automation approach
Phase 1

Design Review & Test Design

Timeline: Early Development

  • Design review
  • Test case creation
  • Environment prep
  • Data planning

Aktivitas Utama:

  • Review technical design
  • Identify testability issues
  • Create detailed test cases
  • Define test data requirements
  • Identify automation opportunities
  • Setup test infrastructure
  • Prepare mock/stub services
Phase 2

Component & Mock Testing

Timeline: Mid Development

  • API contract testing
  • Frontend mock testing
  • Backend unit testing
  • Component validation

Aktivitas Utama:

  • Test API specifications dengan mock
  • Validate request/response format
  • Test error responses
  • Test UI components dengan mock backend
  • Test state management
  • Validate business logic
  • Performance testing
Phase 3

Integration Testing

Timeline: Late Development

  • End-to-end testing
  • Compatibility testing
  • Smoke testing
  • UAT prep

Aktivitas Utama:

  • Test complete user workflows
  • Database integration verification
  • Third-party service integration
  • Cross-browser testing
  • Mobile responsiveness
  • Smoke testing
  • Performance testing
